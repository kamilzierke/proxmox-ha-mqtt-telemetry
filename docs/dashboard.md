# Home Assistant dashboard

The example dashboard is here:

```text
dashboards/proxmox-hardware-dashboard.example.yaml
```

It is a Home Assistant `sections` view using:

- Mushroom cards
- card-mod
- MQTT entities generated by this project
- optional device tracker / link speed entities from the author's setup

## Requirements

Install these Home Assistant frontend integrations first:

- Mushroom Cards
- card-mod

The dashboard uses custom cards such as:

```yaml
type: custom:mushroom-template-card
```

If Mushroom is missing, the dashboard will not render correctly.

## Important limitation

The example dashboard is not fully generic. It contains entity IDs from a real reference setup, for example:

```text
sensor.proxmox_pve_disk_samsung_ssd_980_1tb_s649nu0w940418m_temperature_current
sensor.proxmox_pve_disk_st4000vn006_3cw104_zw60skhl_temperature_current
sensor.proxmox_pve_hardware_proxmox_pve_k10temp_pci_00c3_tctl_temperature
```

Your disk serials, models, node name, or MQTT topic configuration may generate different entity IDs.

Use Home Assistant Developer Tools -> States to find your actual generated entity IDs and adjust the dashboard.

## Known-good view structure

This project uses a full `sections` view YAML structure:

```yaml
type: sections
max_columns: 4
title: hardware
path: hardware
sections:
  - type: grid
    cards:
      - type: markdown
        content: test
cards: []
```

Do not mix this with the indexed internal section representation such as:

```yaml
"0":
  type: grid
  cards:
```

That is a different editor level. Mixing both formats can validate as YAML but render nothing.

## Minimal dashboard test

Before pasting the full dashboard, test this full view:

```yaml
type: sections
max_columns: 4
title: hardware-test
path: hardware-test
sections:
  - type: grid
    cards:
      - type: markdown
        content: |
          # TEST SEKCJI
          If this is visible, full sections YAML works.
      - type: entities
        title: Test entities
        entities:
          - entity: sensor.pve_host_proxmox_link_speed
            name: PVE link speed
          - entity: device_tracker.pve_host_proxmox
            name: PVE host tracker
cards: []
```

If the test works but the full dashboard does not, the problem is likely a missing custom card or a mismatched entity ID.
